# 3,에이전트 테스트 및 공유

## 3.1 Test pane에서 즉시 검증
1. 에이전트 편집 화면에서 **테스트 패널(Test pane)**을 열고 대화를 시작합니다. [1](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-build-agents)[2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  
2. SharePoint/업로드 파일이 **Preparing** 상태면 응답에 반영되지 않을 수 있으니 Ready를 기다립니다. [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  
3. 필요 시 Knowledge 섹션에서 **reload**로 최신 상태를 반영합니다. [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)

# 4) 에이전트 공유(Share) 및 관리

## 4.1 Agent Builder에서 공유하기
1. 에이전트 생성/테스트 후 **Share** 버튼을 선택합니다. [3](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-share-manage-agents)  
2. 공유 대상을 지정합니다(사용자/그룹). [3](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-share-manage-agents)  
3. 공유는 협업/피드백/테스트 목적에 적합하며, 지정 사용자만 접근 가능합니다. [3](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-share-manage-agents)  
4. 문서 기준으로 공유 대상은 **최대 98명**까지 지정할 수 있습니다. [3](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-share-manage-agents)  

---

## 4.2 공유(Sharing) vs 게시(Publishing) 차이(중요)
- Agent Builder의 **공유**는 제한된 사용자에게 직접 접근을 부여하는 방식입니다. [3](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-share-manage-agents)  
- 조직 전체 배포/채널 통합(Teams 등) 같은 정식 배포는 **Copilot Studio에서 Publish**가 필요합니다. [3](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-share-manage-agents)

# 5) 자주 겪는 이슈(Quick Troubleshooting)

## 5.1 업로드/SharePoint 파일이 반영되지 않음
- 신규 파일은 **Preparing** 상태일 수 있으며, 준비까지 몇 분 소요될 수 있습니다. [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  
- Knowledge 섹션의 **reload** 버튼으로 갱신을 시도합니다. [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  

## 5.2 이미지 생성이 테스트에서 안 됨
- 이미지 생성은 **Agent Builder 테스트 패널에서 동작하지 않을 수 있음**이 문서에 명시되어 있습니다. [5](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/image-generator)  

## 5.3 SharePoint를 Knowledge로 못 씀
- 환경/설정에 따라 SharePoint 사용이 제한될 수 있습니다(예: Restricted SharePoint Search 등). [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)

# 6) 실습 과제(옵션)

## 과제 1) 지식 소스 2종 이상 추가
- SharePoint 사이트 1개 + 공용 웹 URL 1개 추가 [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  
- Only use specified sources ON 후, 소스에 없는 질문에서 폴백 확인 [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  

## 과제 2) Teams 채팅 스코핑 검증
- 특정 팀 채널/그룹 채팅 1개 추가(최대 5개까지 가능) [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  
- 해당 채팅에서만 알 수 있는 질문 3개를 설계하고 테스트 [2](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-add-knowledge)  

## 과제 3) 공유 후 피드백 수집
- 동료 3~5명에게 공유(최대 98명) [3](https://learn.microsoft.com/en-us/microsoft-365-copilot/extensibility/agent-builder-share-manage-agents)  
- 피드백 항목: 정확도/근거 제시/톤/누락된 지식 소스/추가 기능 요구
